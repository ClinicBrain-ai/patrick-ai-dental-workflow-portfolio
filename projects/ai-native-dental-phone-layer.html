AI Native Dental Phone Layer

A dentist-in-the-loop mobile workflow concept for reviewing, editing, escalating, or archiving non-final AI-generated drafts.

Project Detail

Problem

AI-assisted workflow drafts need a clear review queue so sensitive text is not treated as final or sent automatically.

Why this matters

Mobile review can make human oversight faster while preserving the boundary between draft support and final human action.

What this project does

It models a local review queue with states for drafted, needs_review, edited, approved_for_manual_send, escalated, and archived.

What this project does NOT do

It does not send messages automatically, integrate with messaging platforms, collect patient data, provide advice, or operate as teledentistry.

Current prototype status

Static portfolio concept with a synthetic queue item and state diagram; no app runtime, messaging integration, backend, or notification system.

Safety Boundary

The project is a workflow-state concept only. It does not collect user data, send messages, automate outreach, or provide medical services.

Human-In-The-Loop Workflow

A human reviewer must approve, edit, escalate, or archive every draft. Approval indicates readiness for manual handling, not automatic delivery.

Portfolio Value

This project shows Patrick's ability to design operational states for safe AI-assisted workflows while keeping sensitive actions under human control.
